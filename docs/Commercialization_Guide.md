# Oral AI 商业化运作与出海实战指南

本文档旨在为 Oral AI 口语练习应用提供从技术开发转向商业运营的完整路径，涵盖合规架构、公司注册、支付打通及产品推广。

## 一、 Stripe 账户简介

Stripe 是全球最主流的互联网支付处理平台，以其对开发者极度友好的 API 和极高的支付成功率著称。

### 为什么选择 Stripe？
1.  **全球收款**：支持 Visa, MasterCard, American Express 等全球主流信用卡。
2.  **本地化支付**：通过海外主体注册的 Stripe，可以一键开启 **支付宝 (Alipay)** 和 **微信支付 (WeChat Pay)** 接口，让您的海外应用能轻松收取国内用户的费用。
3.  **订阅管理**：Stripe Billing 提供了极其强大的订阅管理功能（自动扣费、试用期管理、催款邮件），完美契合 SaaS 产品的会员制模式。
4.  **资金回流**：Stripe 收到的美金可以提现到您的企业银行账户（如 Mercury, Airwallex），再通过合规途径结汇。

## 二、 商业化实战四阶段

### 第一阶段：法律实体与离岸架构 (Foundation)

由于国内主体直接申请 Stripe 和海外 App Store 账号存在诸多限制，**“离岸架构”** 是独立开发者的首选。

#### 1. 注册地选择
*   **首选：美国 (USA)**
    *   **特拉华州 (Delaware)**：适合未来有融资计划的创业公司（C-Corp 架构）。
    *   **怀俄明州 (Wyoming)**：**独立开发者首选**。无州企业所得税，LLC 架构维护成本低，隐私保护好。
*   **备选：英国 (UK)**
    *   成本最低，但银行开户较难（通常需搭配 Wise/Airwallex）。
*   **备选：香港 (HK)**
    *   地缘优势，但维护成本（审计、秘书服务）相对较高。

#### 2. 推荐注册平台
| 平台 | 价格 (参考) | 特点 | 适合人群 |
| :--- | :--- | :--- | :--- |
| **Doola** | ~$297/年 | **最推荐**。专为非美国居民设计，包办 EIN (税号) 申请，与 Mercury 银行合作紧密。 | 不想折腾税务的个人开发者。 |
| **Firstbase.io** | ~$399 (一次性) | **福利最好**。赠送 $5000 AWS 抵扣券等大量 SaaS 优惠，生态资源丰富。 | 技术型团队，能薅羊毛回本。 |
| **Stripe Atlas** | $500 | **最稳健**。Stripe 官方出品，账号秒批，适合融资架构。 | 预算充足，追求绝对稳健者。 |
| **1st Formations** | ~£50 | **最便宜 (英国)**。仅需几十英镑，下证极快。 | 预算极低，仅需跑通流程者。 |

#### 3. 银行账户 (资金流转)
*   **首选**：**Mercury** (美国互联网银行，无实体网点，体验极佳，但开户审核渐严)。
*   **保底**：**Airwallex (空中云汇)** 或 **PingPong**。中国背景的跨境支付公司，对中国法人友好，提供美国虚拟账号，完美支持 Stripe 提现。

### 第二阶段：基础设施与成本模型 (Cost & Pricing)

#### 1. 成本结构
*   **AI 引擎**：Qwen-Omni (DashScope) API 费用。需测算 **单分钟对话成本**。
*   **云资源**：
    *   App Server: 2核4G ECS/EC2 (成本低)。
    *   Storage: COS/S3 存储音频 (成本极低)。
*   **渠道抽成**：
    *   Apple App Store / Google Play: 15% - 30%。
    *   Stripe: ~2.9% + $0.3 每笔。

#### 2. 定价策略 (Pricing)
**原则**：订阅价格应至少为 AI 边际成本的 **3-5 倍**。
*   **Freemium (免费增值)**：
    *   **Free**: 每日 5 分钟限制，仅基础角色。
    *   **Pro ($9.9/月 或 $69.9/年)**：无限畅聊，解锁高级角色 (雅思/商务)，深度纠错报告，云端历史归档。

---

### 第三阶段：支付闭环与技术实现 (Payment Integration)

#### 1. 移动端 (iOS/Android App)
*   **必须使用**：Apple In-App Purchase (IAP) 和 Google Play Billing。
*   **合规警告**：严禁在 App 内包含指向 Web 支付的链接或按钮，否则 100% 拒审。
*   **技术栈**：使用 `react-native-iap` (如果是 RN) 或原生 StoreKit/Billing Library。后端需校验收据 (Receipt Validation)。

#### 2. Web 端 / 落地页 (Web)
*   **核心工具**：**Stripe Checkout** (托管式支付页，无需自己写前端表单，安全合规)。
*   **策略**：
    *   通过邮件营销 (EDM) 或社群，引导老用户在 Web 端续费。
    *   Web 端价格可以比 App 端便宜 20%（因为省去了苹果税），作为用户福利。

---

### 第四阶段：推广与增长 (Growth)

#### 1. 冷启动 (Cold Start)
*   **内容营销**：在 TikTok/小红书/B站发布“AI 模拟雅思口考”或“社恐练口语”的实录视频。**“被 AI 纠正 Chinglish”** 的反差感内容极易传播。
*   **SEO/ASO**：覆盖“雅思口语”、“英语陪练”、“日语会话”等高价值关键词。

#### 2. 增长黑客 (Growth Hacking)
*   **分享裂变**：生成带有“评分/纠错高光”的打卡图片。用户分享解锁 1 天会员。
*   **打卡挑战**：连续打卡 7 天退首月订阅费（沉没成本效应）。
